3. Data Pipeline & Logic Gates

The system operates in four distinct phases to ensure mathematical accuracy and data integrity before any contract is generated.

Step 1: Ingestion & Extraction

Raw URLs are parsed to strip out HTML, leaving pure text. This unstructured text is sent to Groq (Llama 3.3) via a strictly defined JSON schema to extract required financial metrics (Revenue, Gross Margin, Average Daily Balance).

Step 2: Underwriting Mathematics

Extracted variables are passed through strict, deterministic TypeScript logic gates representing OceanX's underwriting policy. Method A (10x ADB) and Method B (20% Revenue Cap) are calculated to output a firm credit recommendation.

Step 3: Master Supervisor Validation

Before any legal document is drafted, the Master Supervisor Agent intercepts the JSON payload. It checks outputs for mathematical integrity and flags missing variables. If data is incomplete (REQUEST INFO), the Supervisor intelligently authorizes a Conditional Contract with TBD placeholders and specific "Conditions Precedent" clauses, while flagging the file for a Human-in-the-loop (HITL) for final verification. This ensures the workflow continues while maintaining absolute accuracy.

Step 4: Context-Aware Document Generation

Authorized by the Supervisor, the Contract Agent acts as an autonomous legal desk. It intelligently adjusts the document type based on data completeness:

  • Scenario A (Full Data): Generates a finalized Commercial Finance Facility Agreement ready for execution.
  • Scenario B (Missing Data): Generates a Conditional Facility Offer & Information Request that explicitly flags missing variables and provides the customer with "Initiation Steps" to connect their live feeds.
This prevents transcription errors and ensures a professional customer experience even for complex edge cases.

4. Evaluation Criteria Alignment & System Implications

  • Master Supervisor Implication (Quality Control): Implementing the Master Agent directly addresses the assignment's "Ideal Architecture" requirement. It acts as a sophisticated oversight layer: rather than simply halting on missing data, it authorizes conditional contract generation with embedded safeguards (TBD placeholders & specific legal requirements). This demonstrates "Practical Usefulness" by keeping the deal flow moving while ensuring that all missing private data is formally requested in the generated document.
  • Clarity of Thinking: Complex back-office tasks are separated into highly specialized agent responsibilities (Financial extraction vs. Legal generation) to prevent hallucinations.
  • Practical Usefulness: Completely eliminates the manual data entry bottleneck between credit approval and contract signing.
  • Scalability: By standardizing the Underwriting output as a JSON payload, this architecture can be instantly scaled to trigger Step 5 (Operations Agent - CIN7/Xero creation) and Step 9 (Collections Agent - GoCardless mandates).

5. Technology Ecosystem & Rationale

The system was built using a high-performance stack designed for the speed and security required by trade finance operations.

Next.js 14 (App Router)

Selected for its high-performance server-side rendering and seamless API route handling, essential for complex agentic handoffs.

Groq & Llama 3.3

The "Decision Brain." We use Groq's low-latency inference to extract financial data from unstructured URLs in milliseconds.

TailwindCSS

Used to craft a premium, "Stripe-class" dark-mode aesthetic that reflects the sophistication of OceanX AI.

TypeScript

Ensures absolute mathematical integrity across the underwriting policy engine and agent communications.

Node.js / Next.js API

The backend orchestration layer. Serverless API routes handle the secure communication between the UI and the agentic logic.

Autonomous Web Engine

A custom-built data acquisition engine that parses raw HTML into structured text, feeding the LLM with clean evidence for decision-making.
